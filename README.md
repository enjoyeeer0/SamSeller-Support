Бот-мост между FunPay и Telegram для поддержки клиентов в одном рабочем интерфейсе.

Подходит для сценария, когда нужно быстро принимать сообщения с FunPay, отвечать прямо из Telegram и держать контроль над диалогами, историей и состоянием системы.

## Возможности

- Пересылка входящих сообщений FunPay в Telegram.
- Ответ клиенту из Telegram:
	- реплаем на уведомление;
	- командой `/to C001 ваш текст`;
	- в режиме активного диалога (`/dialog C001`).
- Поддержка до 3 администраторов одновременно (`TELEGRAM_ADMIN_IDS`).
- Изолированная сессия каждого администратора:
	- отдельный активный диалог;
	- отдельная очистка UI-сообщений;
	- команды одного администратора не ломают интерфейс другого.
- Быстрые действия в уведомлениях:
	- кнопка запуска диалога;
	- кнопка истории за 24 часа.
- Команды мониторинга:
	- `/status`;
	- `/clients`;
	- `/unanswered`;
	- `/history`.
- Антидубли и подавление эхо-сообщений.
- Устойчивость к ошибкам Telegram reply API:
	- fallback, если сообщение для reply уже удалено.
- Файл логов с ротацией (`logs/samseller-support.log`).

## Команды Telegram

- `/help` - помощь по командам.
- `/status` - статус listener и счетчики фильтрации.
- `/clients` - клиенты за последние 24 часа.
- `/unanswered` - клиенты без ответа за последние 24 часа.
- `/history C001 [N]` - история клиента (по умолчанию `N=10`, максимум `50`).
- `/dialog C001` - включить диалог-режим с клиентом.
- `/dialog off` - выключить диалог-режим.
- `/to C001 ваш текст` - отправить сообщение клиенту по коду.

Для обратной совместимости также поддерживается старый ключ `TELEGRAM_ADMIN_ID`.

## Быстрый старт (Windows release)

1. Скачайте `SamSeller-Support-windows-x64.zip` из `Releases`.
2. Распакуйте архив в любую папку.
3. Запустите `SamSeller-Support.exe`.
4. При первом запуске укажите:
	 - FunPay Golden Key;
	 - Telegram Bot Token;
	 - 1-3 Telegram Admin ID.
5. Дождитесь сообщения о запуске моста в логах.

## Логи

- Основной лог: `logs/samseller-support.log`.
- Ротация: до 5 файлов по 5 MB.
- Логируются:
	- запуск потоков;
	- команды администраторов;
	- входящие/исходящие события;
	- фильтрация дублей и эхо;
	- ошибки и восстановление listener.

## Стабильность и защита от сбоев

- Если Telegram не может сделать reply к удаленному сообщению, бот отправит обычное сообщение с клавиатурой (без падения polling).
- При временных сбоях listener продолжает работу с автоматическим перезапуском polling.
- История и коды клиентов хранятся в памяти процесса, есть очистка устаревших данных.

## Ограничения

- Хранилище in-memory: после перезапуска процесса активные сессии и коды клиентов пересоздаются.
- Бот рассчитан на один аккаунт FunPay и до трех Telegram-администраторов.

## Безопасность

- Не публикуйте `secrets.local.json`.
- Не передавайте токен Telegram и Golden Key третьим лицам.

## Структура проекта

- `Main.py` - точка входа.
- `app/config.py` - загрузка и валидация настроек.
- `app/services/funpay_listener.py` - чтение событий и отправка в FunPay.
- `app/services/telegram_bridge.py` - Telegram-команды, UI и маршрутизация сообщений.
- `app/stores/client_store.py` - клиенты, история, unanswered.
- `app/stores/reply_store.py` - привязка Telegram reply к чатам FunPay.
- `app/logging_setup.py` - конфигурация логирования.
